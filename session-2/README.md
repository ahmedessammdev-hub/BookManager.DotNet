# .NET Day 2 Tasks - Book Management Web API Application

A C# Web API application migrating from the Day 1 console application. This project demonstrates framework SDK migration, asynchronous service layers, RESTful controller endpoints, custom request logging middleware, and custom domain exceptions with ASP.NET Core.

---

## Project Structure

The project has been migrated and structured as a Web API:

```text
session-2/
│
├── Controllers/          # API Controllers
│   └── BooksController.cs # REST endpoint exposing the book resource list
│   
├── Exceptions/           # Custom Domain Exceptions
│   └── BookNotFoundException.cs # Custom exception for missing books
│   
├── Middleware/           # HTTP Request Pipeline Middleware
│   └── RequestLoggingMiddleware.cs # Custom middleware logging HTTP requests and response speed
│   
├── Entities/             # Domain Model Entities (from Day 1)
├── Data/                 # Data Layer (from Day 1)
├── Services/             # Business Logic Layer (Asynchronous)
│   ├── IBookService.cs   # Async contract for Book CRUD operations
│   └── BookService.cs    # Async implementation using Task/Task<T>
│   
├── Program.cs            # API configuration, middleware pipeline, and host run definition
└── Session2.csproj       # Web SDK configuration file (.NET 10.0)
```

## Task Implementations

### Web API Migration (Task 2.1)
- The project SDK in [Session2.csproj](./Session2.csproj) was updated to `Microsoft.NET.Sdk.Web` to enable web hosting features.
- Swashbuckle OpenAPI tooling is centrally managed via `Directory.Packages.props` and registered to generate interactive OpenAPI/Swagger documentation.
- The C# JSON serializer is configured with `ReferenceHandler.IgnoreCycles` in [Program.cs](./Program.cs) to prevent infinite circular reference loops during serialization (as `Book` references `Author` which back-references `Book`).

### Asynchronous Refactoring (Task 2.2)
To simulate database connections in production environments, the business service layer was updated to run asynchronously:
- Converted [IBookService.cs](./Services/IBookService.cs) to use C# task-based asynchronous structures returning `Task` or `Task<T>`.
- Appended `Async` suffixes to method names (e.g., `GetAllAsync`, `GetByIdAsync`, `CreateAsync`).
- Refactored [BookService.cs](./Services/BookService.cs) utilizing `Task.FromResult` for in-memory list operations and `Task.CompletedTask` for asynchronous void operations.

### Books REST Controller (Task 2.3)
- Implemented [BooksController.cs](./Controllers/BooksController.cs) inheriting from `ControllerBase` and decorated with the `[ApiController]` and route attribute configurations.
- Utilizes ASP.NET Core Constructor Dependency Injection to resolve `IBookService`.
- Exposes `GET /api/books` returning a HTTP `200 OK` status with the complete list of seeded book models.

### Request Logging Middleware (Task 2.4)
- Implemented a custom [RequestLoggingMiddleware.cs](./Middleware/RequestLoggingMiddleware.cs) to measure request pipeline speeds and log metadata.
- Instantiates a `Stopwatch` on incoming requests to measure execution duration in milliseconds.
- Formats logs indicating HTTP verb, target path, HTTP status code, and execution time:
  ```text
  info: tasks.Middleware.RequestLoggingMiddleware[0] HTTP GET /api/books responded 200 in 83 ms
  ```

### Custom Domain Exceptions (Task 2.5)
- Created [BookNotFoundException.cs](./Exceptions/BookNotFoundException.cs) representing a missing domain model resource.
- Updated `BookService.GetByIdAsync` to check lookup results and throw `BookNotFoundException` when a book with the specified ID is not present in the store.

## How to Run and Test

### Prerequisites
- [.NET SDK 8.0 or 10.0](https://dotnet.microsoft.com/download) installed on your system.

### Build and Run
From the workspace root directory, run:
```bash
# Start the Web API server
dotnet run --project session-2
```

### Accessing Swagger & Testing the API
1. Once the application starts, open your browser and navigate to:
   **`http://localhost:5005`**
2. The interactive Swagger UI will load directly, displaying the exposed `GET /api/books` endpoint.
3. Click **Try it out** and then **Execute** to fetch the list of seeded books.
4. Check your terminal/console logs to see the output generated by the `RequestLoggingMiddleware`.
